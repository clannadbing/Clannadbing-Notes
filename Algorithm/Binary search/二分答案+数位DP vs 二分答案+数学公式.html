二分答案+数位DP vs 二分答案+数学公式

3007. 价值和小于等于 K 的最大数字

解题思路

二分答案 + 数位 DP

由于 num 越大，价值和也越大，有单调性，可以二分答案。

于是问题转换成：计算从 1 到 num 的价值和，判断其是否 ≤k。

思路和 233. 数字 1 的个数 是一样的，原理 题解 或者视频讲解 数位 DP 通用模板 v1.0（本题没有下界约束，用 v1.0 就够了）。

设 num 的二进制长度为 n，我们只需要在 题解 的基础上，额外在 d=1 时判断当前下标是否为 x 的倍数，如果是，把 cnt加一。

最后还剩一个问题：二分的上界取多少合适？

1 到 num 中的数字 v，除以 2x−1 后如果是奇数，就说明 v 至少包含一个我们需要的 1。而 1 到 num 中有 ⌊(num+1)/2⌋ 个奇数，所以取 num 为(k+1)⋅2x可以保证价值和至少是 k+1，一定不满足二分判定。

代码中用到的位运算技巧见 从集合论到位运算，常见位运算技巧分类总结！

二分答案+数学公式

举例说明，1 到 num=6 的二进制表示如下：

001010011100101110

考虑计算 1 到 num 的每个比特位上有多少个 1。

最低位：相当于计算 1 到 num 中有多少个奇数，这有 ⌊num+12⌋=3 个。

次低位：我们可以把每个数都除以 2（右移 1 位），忽略 0，得到：

0101101011

同样地，考虑其中的奇数，发现 1 到 ⌊num2⌋−1 中的每个奇数都出现了 2 次，但是 ⌊num2⌋ 只出现了 1 次。所以次低位中有 3 个 1。

从右往左第三位：我们可以把每个数都除以 22=4（右移 2 位）。同样地，考虑其中的奇数，发现 1 到 ⌊num4⌋−1中的每个奇数都出现了 4 次，但是 ⌊num4⌋=1出现了 3 次。这个 3 怎么算？我们可以把 num=6 和 22−1=3取 AND，得到 2，说明把 ⌊num4⌋作为前缀的二进制数，最低位和次低位是 00,01,10，这有 2+1=3 个。

一般地，从低到高第 i 位（i 从 0 开始）的 1 的个数，我们分两部分计算：

  1. 设 ⌊num2i⌋⌋，从 1 到 n−1 有 ⌊n2⌋个奇数，每个奇数作为 1 到 num 的二进制数的前缀，出现了 2i 次。所以一共有 ⌊n2⌋⋅2i 个 1。
  2. n 单独作为 1 到 num 的二进制数的前缀，出现了 num&(2i−1))+1 次。如果 n 是奇数，那么它为第 i 位贡献了 num&(2i−1))+1个 1，否则为第 i 位贡献了 0 个 1。 最后，按照题目要求，只统计 x 的倍数位置上的 1 的个数。

//方法一
class Solution {
public:
    long long findMaximumNumber(long long k, int x) {
        function<long long(long long)> check = [&](long long num) {
            string str;
            while (num != 0) {
                if (num & 1) str = "1" + str;
                else str = "0" + str;
                num >>= 1;
            }
            int n = str.size();
            vector<vector<long long>> memo(n, vector<long long>(n+1, -1));
            function<long long(int, int, bool)> dfs = [&](int i, int cnt, bool isLimit) -> long long {
                if (i == n) {
                    return cnt;
                }
                if (!isLimit && memo[i][cnt] != -1) {
                    return memo[i][cnt];
                }
                int up = isLimit ? str[i] - '0' : 1;
                long long res  = 0;
                for (int j = 0; j <= up; j++) {
                    res += dfs(i+1, (j == 1 && (n - i) % x == 0) ? cnt+1:cnt, isLimit && j == up); 
                }
                if (!isLimit) {
                    memo[i][cnt] = res;
                }
                return res;
            };
            return dfs(0, 0, true);
        };
        long long l = 0;
        long long r = (k + 1) << x;
        while (l <= r) {
            long long mid = l + (r - l) / 2; 
            if (check(mid) <= k) {
                l = mid + 1;
            }
            else {
                r = mid - 1;
            }
        }
        return r;
    }
};

class Solution {
public:
    long long findMaximumNumber(long long k, int x) {
        auto check = [&](long long num) {
            long long res = 0;
            int i = x - 1;
            for (long long n = num >> i; n; n >>= x, i += x) {
                res += (n / 2) << i;
                if (n % 2) {
                    long long mask = (1LL << i) - 1;
                    res += (num & mask) + 1;
                }
            }
            return res <= k;
        };


        // 开区间二分，原理见 https://www.bilibili.com/video/BV1AP41137w7/
        long long left = 0, right = (k + 1) << x;
        while (left + 1 < right) {
            long long mid = left + (right - left) / 2;
            (check(mid) ? left : right) = mid;
        }
        return left;
    }
};

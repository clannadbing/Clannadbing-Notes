中位数贪心+裴蜀定理

6329. 使子数组元素和相等

前置知识-神兵利器 nth_element

头文件：#include

nth_element:在数组或容器中将第几大或小的元素放到该放的位置上。（默认第几小，可以用cmp自定义为第几大）

如：nth_element(a,a+6,a+10); //在a中，找到 第七小 将其放到a[6].

但是值得注意的是：a[6]左边、右边的元素不一定有序！！

解题思路

为了方便描述，将 arr 简记为 a

提示1

先来解决 a 不是循环数组的情况

根据题意，考虑从 i 和 i+1 开始的两个长为 k 的子数组的和，如果要求这两个和相等，则有

a[i]+a[i+1]+...+a[i+k−1]=a[i+1]+a[i+2]+...+a[i+k]

化简得

a[i]=a[i+k]

换句话说

a[0]=a[k]=a[2k]=...a[1]=a[k+1]=a[2k+1]=...a[2]=a[k+2]=a[2k+2]=......

提示2

按照 i mod k 的结果将 a 分组，对每一组（记作 b），我们需要解决：让数组 b 的所有元素相等的最少运算次数。

根据中位数贪心，将 b 的所有元素变为 b 的中位数是最优的。

证明：设 b 的长度为 m，设要将所有 b[i] 变为 x。假设 b 已经从小到大排序。首先，如果 x 取在区间[b[0],b[m−1]] 之外，那么 x 向区间方向移动可以使距离和变小；同时，如果 x 取在区间 [b[0],b[m−1]] 之内，无论如何移动 x ，它到 b[0] 和 b[m−1] 的距离和都是一个定值 b[m−1]−b[0], 那么去掉 b[0] 和 b[m−1]这两个最左最右的数，问题规模缩小。不断缩小问题规模，如果最后剩下 1个数，那么 x 就取它；如果最后剩下 2 个数，那么 x 取这两个数之间的任意值都可以（包括这两个数）。因此 x 可以取 b[m/2]。

提示3

回到原问题。

比如 n=6,k=4，那么 a[2] 循环后 a[8] ，和 a[0] 在同一组，而 a[1] 无论怎么循环都无法和 a[0] 在同一组。（(1+6n)mod4≠0）

根据这个例子，可以猜想一个结论：

一个循环数组如果既有周期 n，又有周期 k，则必然有周期 gcd(n,k)

证明：根据 裴蜀定理，有

a[i]=a[i+nx+ky]=a[i+gcd(n,k)]

这样就转换成了不是循环数组的情况。

实现

class Solution {
public:
    long long makeSubKSumEqual(vector<int>& arr, int k) {
        int n = arr.size();
        k = gcd(n,k);
        vector<vector<int>> res(k);
        long long ans = 0;
        for (int i = 0; i < arr.size(); i++) {
            res[i % k].push_back(arr[i]);
        }
        for (int i = 0; i < res.size(); i++) {
            nth_element(res[i].begin(), res[i].begin() + res[i].size()/2, res[i].end());
            for (int j = 0; j < res[i].size(); j++) {
                ans += abs(res[i][j]-res[i][res[i].size()/2]); 
            }
        }
        return ans;
    }
};

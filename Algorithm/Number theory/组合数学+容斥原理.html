给小朋友们分糖果 II-解题思路

要计算合法方案数（每个小朋友分到的糖果都不超过 limit），可以先计算所有方案数（没有 limit 限制），再减去不合法的方案数（至少一个小朋友分到的糖果超过 limit）。

所有方案数

相当于把 n 个无区别的小球放入 3 个有区别的盒子，允许空盒的方案数。

隔板法：假设 n 个球和 2 个隔板放到 n+2 个位置，第一个隔板前的球放入第一个盒子，第一个隔板和第二个隔板之间的球放入第二个盒子，第二个隔板后的球放入第三个盒子。那么从 n+2 个位置中选 2 个位置放隔板，有 C(n+2,2) 种放法。注意隔板可以放在最左边或最右边，也可以连续放，对应着空盒的情况。

至少一个小朋友分到的糖果超过 limit

设三个小朋友分别叫 A,B,C。

只关注 A。如果 A 分到的糖果超过 limit，那么他至少分到 limit+1 颗糖果，问题变成剩下 n−(limit+1) 颗糖果分给三个小朋友的方案数（注意 B 和 C 分到的糖果是否超过 limit 我们是不关注的），即 C(n−(limit+1)+2,2)。

只关注 B 的情况和只关注 C 的情况同上，均为 C(n−(limit+1)+2,2)。

直接加起来，就是 3⋅C(n−(limit+1)+2,2)，但这样就重复统计了「至少两个小朋友分到的糖果超过 limit」的情况，要减去。

至少两个小朋友分到的糖果超过 limit

只关注 A 和 B。如果他们俩分到的糖果超过 limit，那么至少分出去了 2⋅(limit+1)颗糖果，问题变成剩下 n−2⋅(limit+1)颗糖果分给三个小朋友的方案数（注意 C分到的糖果是否超过 limit 我们是不关注的），即 C(n−2⋅(limit+1)+2,2)。

只关注 A,C 的情况和只关注 B,C 的情况同上，均为 C(n−2⋅(limit+1)+2,2)。

直接加起来，就是 3⋅C(n−2⋅(limit+1)+2,2)，但这样就重复统计了「三个小朋友分到的糖果均超过 limit」的情况，要减去。

三个小朋友分到的糖果超过 limit

至少分出去了 3⋅(limit+1) 颗糖果，问题变成剩下 n−3⋅(limit+1) 颗糖果分给三个小朋友的方案数，即 C(n−3⋅(limit+1)+2,2)。

总结

不合法的方案数为「至少一个」减去「至少两个」加上「三个」，这就是容斥原理。

最后用所有方案数减去不合法的方案数，整理得到答案：

C(n+2,2)−3⋅C(n−limit+1,2)+3⋅C(n−2⋅limit,2)−C(n−3⋅limit−1,2)

// 枚举
class Solution {
public:
    long long distributeCandies(int n, int limit) {
        long long res = 0; 
        for (int i = 0; i <= limit; i++) {
            int num = n - i;
            if (num > 2*limit) {
                continue;
            } 
            else if (num > limit && num <= 2*limit) {
                res = res + (num + 1) - 2*(num - limit);
            }
            else if (num >= 0 && num <= limit) {
                res = res + num + 1; 
            }
            else {
                continue;
            }
        }
        return res;
    }
};

class Solution {
public:
    long long C2(int n) {
        if (n < 2) {
            return 0;
        }
        return 1LL * n * (n-1) / 2;
    }
    long long distributeCandies(int n, int limit) {
        return C2(n+2) - 3*C2(n-limit+1) + 3*C2(n-2*limit) - C2(n-3*limit-1);
    }
};

重新排列后包含指定子字符串的字符串数目-解题思路

正难则反。总共有 26n个字符串，减去不含 leet 的字符串个数，就得到了答案。

不含 leet 的字符串，需要至少满足如下三个条件中的一个：

  1. 不含 l。
  2. 不含 t。
  3. 不含 e 或者恰好包含一个 e。

分类讨论。

至少满足一个条件

  1. 不含 l：每个位置可以填 25 种字母，方案数为 25n 。
  2. 不含 t：同上，方案数为 25n。
  3. 不含 e 或者恰好包含一个 e：不含 e 同上，方案数为 25n ；恰好包含一个 e，先从 n 个位置中选一个填 e，然后剩下 n−1 个位置不能包含 e，方案数为 n⋅25n−1 。加起来就是 25n+n⋅25n−1 。

直接加起来，就是 (3⋅25+n)⋅25n−1，但这样就重复统计了「至少满足两个条件」的情况，要减去。

至少满足两个条件

  1. 不含 l 和 t：每个位置可以填 24 种字母，方案数为 24n。
  2. 不含 l 且 e 的个数不足两个：同「满足一个条件」中 3 的分析，额外不能填 l，方案数为 24n+n⋅24n−1 。
  3. 不含 t 且 e 的个数不足两个：同上，方案数为 24n+n⋅24n−1 。

直接加起来，就是 (3⋅24+2n)⋅24n−1，但这样就重复统计了「满足三个条件」的情况，要减去。

满足三个条件

同「满足一个条件」中 3 的分析，额外不能填 l 和 t，方案数为 23n+n⋅23n−1 。

总结

不含 leet 的字符串的个数为「至少满足一个条件」减去「至少满足两个条件」加上「满足三个条件」，这就是容斥原理。

最后用 26n 减去不含 leet 的字符串的个数，得到答案：

26n−(3⋅25+n)⋅25n−1+(3⋅24+2n)⋅24n−1−(23+n)⋅23n−1

其中 xn 可以用快速幂计算，具体请看 50. Pow(x, n)。

// 容斥原理
class Solution {
public:
    long long pow(long long x, int n) {
        int MOD = 1e9 + 7;
        long long res = 1;
        while (n != 0) {
            if (n & 1) {
                res = res * x % MOD;
            } 
            x = x * x % MOD;
            n = n >> 1;
        }
        return res;
    }
    int stringCount(int n) {
        int MOD = 1e9 + 7;
        int res = pow(26, n) % MOD;
        res = res - (75+n) * pow(25, n-1) % MOD;
        res = res + (72+2*n) *pow(24, n-1) % MOD;
        res = res - (23+n)*pow(23, n-1) % MOD;
        return (res % MOD + MOD) % MOD;
    }
};

// 看成是「至少装满型」分组背包，每组都可以从 a 到 z 中选一个物品，求至少有 1 个 l、1 个 t 和 2 个 e 的方案数。
class Solution {
public:
    int stringCount(int n) {
        int MOD = 1e9 + 7;
        long long memo[n+1][2][2][3];
        memset(memo, -1, sizeof(memo));
        function<long long(int, int, int, int)> dfs = [&](int n, int l, int t, int e) -> long long {
            if (n == 0) {
                if (l == 0 && t == 0 && e == 0) {
                    return 1;
                }
                else {
                    return 0;
                }
            }
            if (memo[n][l][t][e] != -1) {
                return memo[n][l][t][e];
            }
            long long res = 0;
            res = res + dfs(n-1, max(l-1, 0), t, e);
            res = res + dfs(n-1, l, max(t-1, 0), e);
            res = res + dfs(n-1, l, t, max(e-1, 0));
            res = res + 23 * dfs(n-1, l, t, e);
            memo[n][l][t][e] = res % MOD;
            return memo[n][l][t][e];
        };
        return dfs(n, 1, 1, 2);
    }
};

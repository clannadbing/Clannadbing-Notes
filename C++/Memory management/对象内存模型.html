虚拟内存模型

在学习对象内存模型之前，简单介绍一下操作系统的虚拟内存模型，无论是32位还是64位OS，它们的虚拟内存模型布局都是相同的，均分为内核区、栈区、共享内存区、堆区、静态存储区、代码区，我们从32位OS入手学习操作系统虚拟内存模型。32位OS的内存布局如下：

 

所有进程的虚拟内存分为两部分——内核区和用户区，图中最上部分的区域就是内核区，下方所有的地址空间则都属于用户区，通常各大平台介绍的所谓的“C++内存模型”指的就是操作系统虚拟内存用户区内存布局。用户区可以分为：栈区、共享内存区、堆区、静态存储区、代码区，它们的用途如下：

  • 栈空间：用于保存不是通过new，malloc实例化的变量，比如int a = 1、函数形参都存储在栈上；
  • 共享内存空间：用于进程之间通信；
  • 堆空间：存储通过new，malloc实例化的变量；
  • 静态存储区：分为BSS区和Data区，BSS区存储运行时还未初始化的静态变量，Data区存储运行时已经初始化的变量；
  • 代码区：存储逻辑代码。

对象内存模型

不同编译器编译对象的方式不同，因而对象的内存布局也会不同，换句话说，C++对象内存布局方式由编译器决定，如果我们不深究编译器，是否有必要学习对象内存模型呢？答案是肯定的，即使不学习编译器原理，学习对象内存模型有助于帮助我们更好地理解一些C++知识点，比如：trivial、standard_layout类型，RTTI实现原理，多态和继承的原理...

普通对象

C++对象成员可以分为两类：成员函数和成员属性。成员函数可以分为：构造函数、析构函数、运算符重载函数，虚成员函数，静态成员函数、普通成员函数、内联成员函数；成员属性可以分为：普通成员属性，静态成员属性。

class Object
{
public:
    Object(){}//构造函数
    Object(Object &obj){/**省略**/}
    Object &operator=(Object &obj){/**省略**/}//运算符重载函数
    ~Object(){}//析构函数
    inline void inlineFunction(){}//内联函数
    void ordinaryFunction(){}//普通成员函数
    virtual void virtualFunction(){}//虚成员函数
    static void staticFunction(){}//静态成员函数
private:
    int m_attribute;//普通成员属性
    static int m_staticAttribute;//静态成员属性
};

这些成员在内存中的存放方式如下：

  • 所有成员函数都存放在代码区，其中除了虚函数，其它所有成员函数地址通过编译器获取；
  • 虚函数存放在代码区，与其它成员函数不同的是，虚函数地址存放在与该对象关联的虚表中；
  • 静态成员变量存放在静态存储区；
  • 普通成员变量存放在对象内，遵循字节对齐原则存放。

为什么要把对象内存布局设计成这种方式呢？接下来对上述4种存放方式进行说明。

第一点

第三点

静态成员属性存放在静态存储区。这是因为静态成员属性通过类名访问，而不是通过对象名访问，因此不可能存放在对象中。

第四点

普通成员属性存放在对象中，这点没什么好说的。简单提一下后面一句话，普通成员变量在对象中是以字节对齐的方式存放的，字节对齐规则如下：

编译器提供了手动指定对齐值的关键字 #pragma pack(N)，可以手动设置对齐的字节数，比如 #pragma pack(1)、#pragma pack(4)等。32位系统默认为4，64位系统默认为8。

  1. 将各数据成员内存对齐。 对齐模数是【该数据成员所占内存】与【#pragma pack指定的数值】中的较小者。数据成员所占内存应该是该模数的整数倍；另外，如果对齐模数为 N ，存放的起始位置就是 N 的整数倍；
  2. 将结构体整体内存对齐。 对齐模数是【结构体内部最大的基本数据类型成员】、【#pragma pack指定的数值】长度中数值较小者。结构体所占内存以及起始位置应该是该模数的整数倍；
  3. 结构体嵌套结构体时，对齐模数是【子结构体中数据类型最大值】 和 【#pragma pack指定的数值】 中的较小值。结构体成员所占内存以及起始位置应该是该模数的整数倍。

trivial类型对象

trivial类型是一种特殊的对象类型，它的内存布局和C语言中的结构体相同，之所以把它从各种对象中单独提取出来，是出于优化的考虑，我们对这类对象进行构造、析构、拷贝和赋值时可以采用更有效率的方法——C语言中的memcpy函数。

什么样的对象满足trivial类型？

  • 成员函数：首先构造函数、析构函数、重载运算符函数只能是编译器创建的默认函数，第二不能有虚函数；
  • 成员属性：如果成员属性有对象类型，该对象也需要是trivial类型；
  • 继承：如果对象有父类，父类需要是trivial类型，且继承方式不能是虚继承。

为什么要满足上述三点要求？我们可以从C++对象内存模型的角度解释原因：

第一点，为什么不允许虚函数存在，C语言结构体内不存在虚表指针，因此trivial类型不能有虚函数；为什么trivial类允许有C语言中没有的成员函数，成员函数存放在代码区，并不占用对象内存；为什么不允许自定义构造函数、析构函数、重载运算符函数，编译器正是根据我们是否定义了这些函数来判断该类是否属于trivial类型，如果属于trivial类型，编译器会为这个类生成效率更优的构造函数、析构函数、重载运算符函数。

第二点，为什么成员属性只能是trivial类型，成员属性存放在对象空间，如果成员属性不能通过memcpy赋值，那么这个对象自然不能通过memcpy赋值。

第三点，为什么继承方式不能是虚继承，原因和不允许有虚函数类似，如果有虚继承，就会有虚表，有虚表，对象内就会有虚表指针，而C语言结构体内不存在虚表指针。

访问权限

访问权限关键字private，public，protected是否影响对象的内存布局呢？结论是影响，这些关键字的功能除了编译的时候检测用户编写的代码是否越权访问，如果越权则编译不通过，还会影响内存布局。
